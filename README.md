# LLM application

Комплексное решение на основе языковых моделей, включающее чат-бота для электронной коммерции и систему индексации документов с векторным поиском.

## Обзор проекта

Проект состоит из двух основных компонентов:

### E-commerce Chatbot 
Интеллектуальный чат-бот для обслуживания клиентов, построенный на LangChain и интегрированный с OpenRouter API.

### PDF Indexing System 
Система для обработки PDF документов, извлечения текста (включая OCR), создания векторных представлений и организации семантического поиска через Qdrant.


### E-commerce Chatbot
- **Умная обработка запросов**: Использование современных языковых моделей для понимания и ответов на вопросы клиентов
- **FAQ система**: Автоматический поиск ответов в базе часто задаваемых вопросов
- **Отслеживание заказов**: Проверка статуса заказов по номеру через команду `/order`
- **История диалога**: Сохранение контекста разговора для более естественного общения
- **Логирование**: Ведение журнала всех сессий для анализа и отладки
- **Русскоязычный интерфейс**: Полностью локализованный для русскоязычных пользователей

### PDF Indexing System
- **Многопоточное сканирование**: Обработка больших объемов PDF документов
- **Извлечение текста**: Поддержка как текстового слоя, так и OCR (русский и английский)
- **Интеллектуальное разбиение**: Чанкинг текста с перекрытием для сохранения контекста
- **Векторизация**: Использование Sentence Transformers для создания эмбеддингов
- **Семантический поиск**: Поиск по смыслу через Qdrant векторную базу данных
- **Метаданные**: Автоматическое извлечение авторов, года публикации, DOI и другой информации

## ️ Установка

1. Клонируйте репозиторий:

2. Создайте виртуальное окружение:

## Использование
### E-commerce Chatbot
**Запуск основного бота:**
```bash
python ecom_bot/app.py
```

**Доступные команды**
- Общение: Просто задайте вопрос боту на русском языке

- Проверка заказа: `/order 12345` (где 12345 - номер заказа)

- Очистка контекста: сброс` - очищает историю диалога

- Выход: `выход, стоп или конец` - завершение сессии

**Пример использования**
```
Чат-бот запущен! Можете задавать вопросы. 
 - Для выхода введите 'выход'.
 - Для очистки контекста введите 'сброс'.

Вы: Как оформить возврат?
Бот: Вы можете оформить возврат в течение 14 дней. Сохраните чек и упаковку.

Вы: /order 12345
Бот: Заказ #12345: доставлен

Вы: сброс
Бот: Контекст диалога очищен.
```



### PDF Indexing System
**Подготовка документов:**

1. Поместите PDF файлы в директорию ```data/```
2. Убедитесь, что Qdrant запущен (```docker-compose up -d```)

**Запуск индексации:**
```bash
python src/build_index.py
```

**Пример вывода индексации:**
```text

Успешно:            11
Ошибок:             1
Создано векторов:   156
```

**Запуск поиска:**
```bash
python src/rag_chain.py
```

**Пример вывода поиска:**

**Запуск интерфейса в браузере:**
```bash

```
## Конфигурация
**Настройка E-commerce Bot**

***FAQ система***

Отредактируйте файл ```ecom_bot/data/faq.json``` для добавления или изменения часто задаваемых вопросов:

```json
[
  {
    "q": "Ваш вопрос?",
    "a": "Ответ на вопрос"
  }
]
```

***Настройка заказов***

Отредактируйте файл ```ecom_bot/data/orders.json``` для управления базой заказов:

```json
{
  "12345": {
    "status": "доставлен"
  }
}
```

**Настройка PDF Indexing System**

***Параметры индексации (в ```src/build_index.py```):***

- TextChunker(size=800, overlap=150) - размер чанков и перекрытие
- LocalEmbedder(model="sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2") - модель эмбеддингов
- QdrantUploader(collection_name="papers_index") - название коллекции

***Настройка OCR:***
```python
parser = PdfParser(use_ocr=True)  # Включить/выключить OCR
```

***API и компоненты***

****Основные классы PDF системы****
- PdfScanner: Сканирование директории с PDF файлами
- PdfParser: Извлечение текста и метаданных из PDF
- TextChunker: Интеллектуальное разбиение текста на чанки
- LocalEmbedder: Создание векторных представлений
- QdrantUploader: Загрузка векторов в Qdrant
- VectorStore: Обертка для работы с Qdrant API
- IndexBuilder: Оркестратор всего процесса индексации
